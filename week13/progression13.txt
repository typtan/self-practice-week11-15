
ในคลาสนี้ได้เรียนเรื่อง Date time หนูเลยได้ฝึกการใช้งาน Date object พื้นฐาน เพื่อให้เข้าใจก่อนว่าการทำงานกับเวลาใน javaScript จริงๆแล้วมีหลายรูปแบบ 
ทั้งเวลาปัจจุบัน เวลาที่สร้างจาก millisecond และเวลาที่สร้างจาก date string และได้รู้ว่าการเขียน new Date() เฉยๆ ทำให้หนูได้เรียนรู้ว่า
เวลาที่ได้เป็นค่าแบบสากล (utc) ไม่ใช่เวลาที่ไทย ตัว console.log(today.getTime()) ก็ช่วยให้หนูเข้าใจว่าเวลาทุกอย่างจริงๆ ถูกเก็บเป็นตัวเลข millisecond 

และหนูก็ได้นำเรื่องเวลามาปรับใช้พัฒนา project ใน sprint 3 และมีการเขียน changePlan ตัว changeBtn หนูเขียนให้เริ่มจาก default behavior แล้วตรวจสอบว่า user เลือกแผนหรือยัง 
จากนั้นเรียก changePlan ซึ่งเป็นฟังก์ชันที่ไปยิง PUT api และหนุก็ได้เริ่มได้เรียนรู้รูปแบบการจัดการผลลัพธ์จาก backend ผ่านพวก status code ไม่ใช่แค่ json อย่างเดียว 
เช่น ถ้า res.status === 200 หมายถึง update สำเร็จ ก็จะแสดง dialog และ refresh สถานะใหม่และใช้เรื่องเวลาที่เราได้เขียนไว้มาใช้ด้วย 
แต่ถ้าเป็น 404 ก็หมายถึงไม่มีแผนให้เปลี่ยน database ไม่มีข้อมูลน่าจะยังไม่เคยลงแล้วก็ได้มีการเขียน handle status 409 ที่เป็นสถานะที่ backend ใช้เพื่อบอกว่าเกิดข้อขัดแย้งกัน 
เช่น user พยายามเปลี่ยนเป็นแผนเดิมอีกครั้ง หรืออยู่ในสถานะที่ไม่อนุญาตให้เปลี่ยน 

โดยในเรื่องของเวลา updatedAt เพื่อที่จะใช้แสดงข้อความในหน้า UI หนุได้ดึง timezone จริงของผู้ใช้และแปลง data.updatedAt 
ซึ่งเป็น utc จาก backend ให้เป็นเวลาใน timezone ที่ถูกต้องด้วย new Date().toLocaleString() และ formatting แบบ en-GB
