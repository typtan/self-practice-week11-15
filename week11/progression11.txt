
เริ่มจากที่หนูเขียน function ง่ายๆตามอาจารย์เพื่อ GET ข้อมูลที่ใช้ fetch แล้วเช็ค res.ok ถ้าไม่โอเคก็ให้มันโยน error ออกมา 
จากนั้นก็แปลงข้อมูลที่ได้เป็น JSON แล้ว return กลับไป และทำให้หนูได้เข้าใจการใช้ async / await , การเช็ค res.ok 
และการแปลง JSON มากขึ้น

และต่อมาหนูก็ได้เอาที่อาจารย์สอนไปพัฒนา project กับเพื่อนๆในกลุ่ม โดยใช้ fetch ดึงข้อมูลจาก api เช่น fetchStudyPlans() 
และมีการใช้ fetch กับ endpoint จริง และใช้การรับมือกับ error ด้วยการเช็ค res.ok พร้อม status code ทำให้เริ่มเข้าใจว่างานจริงๆต้องจัดโครงสร้างยังไงให้สะอาด แล้วก็สามารถนำไปใช้ซ้ำในหลายๆหน้าได้
แล้วก็มีฟังก์ชันที่ซับซ้อนขึ้นก็คือ getDeclaredPlan(studentId) เพราะต้องจัดการเรื่อง authentication ด้วย token จาก Keycloak ก่อนจะ fetch แล้วก็ได้เข้าใจว่า api จริงในระบบที่มี security ไม่เหมือนกับ fetch แบบลองเล่นๆตอนแรก 
เพราะต้องเพิ่มขั้นตอนพวก token และรับมือกับปัญหา เช่น ถ้า api คืน 404 มันคือการที่นศ.คนนี้ยังไม่เคย declare plan 
ดังนั้นต้อง return null ไม่ใช่โยน error ทำให้หนูได้เรียนรู้การเขียน business logic เฉพาะของระบบลงไปด้วย
